## Core Principle

**Curate, don't create.**

Decades of software engineering have produced battle-tested principles:
- Clean Code (Robert Martin)
- SOLID principles
- Unix Philosophy
- Standard linters and formatters

TEOF's role: Select which tools align with observation primacy. Not reinvent them.

---

## Recommended Stack

```
Linter:      Language-standard with strict rules
Formatter:   Language-standard (no debates)
Tests:       Language-standard framework
Principles:  Clean Code + SOLID + Unix Philosophy
```

TEOF configuration = selecting which rules to enable, not writing new rules.
