# Source Framework Analyses

This directory contains analyses of existing frameworks that have survived reality-testing in their domains.

## Purpose

Before borrowing from a framework, understand:
1. Why has it persisted?
2. What selection pressures has it survived?
3. What observable outcomes does it produce?
4. What's core (stable) vs peripheral (adaptive)?
5. Where might it conflict with other frameworks?

## Template

Each analysis should follow this structure:

```markdown
# [Framework Name] Analysis

**Domain:** [What domain does it address?]
**Track Record:** [How long? Under what conditions?]
**Observable Outcomes:** [What do practitioners demonstrate?]

---

## Why It Persists

[Analysis of what makes it robust]

## Core vs Peripheral

**Core (stable, rarely changes):**
- [Element 1]
- [Element 2]

**Peripheral (adapts to context):**
- [Element 1]
- [Element 2]

## Selection Pressures Survived

[What has tried to kill it and failed?]

## Observable Outcomes

[What can you see in practitioners that non-practitioners lack?]

## Integration Notes

**Aligns with:**
- [Other framework] on [specific point]

**Conflicts with:**
- [Other framework] on [specific point]
- Resolution: [How to handle]

## What to Borrow

[Specific elements worth integrating]

## What to Leave

[Elements that are context-specific or don't transfer]

---

**Analysis Date:** [Date]
**Analyst:** [Human/AI-assisted]
**Verification:** [How was this checked against reality?]
```
