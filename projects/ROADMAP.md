## Strategic Pivot (2025-11-30)

**Previous path:** Content → Audience → Product (12-24 month grind)

**New path:** Protocol launch → Services layer → Network growth

**Core reframe:** TEOF is not a product to own. It's a protocol for decentralized coherence — like Bitcoin for truth-finding.

**Why:**
- Validates faster (days vs months)
- More leveraged (build once vs create daily)
- More differentiated (novel vs crowded)
- Mission-aligned (system, not personality)
- Higher ceiling (protocol adoption vs company revenue)
- Anti-fragile to forking (incoherent forks die; coherent forks strengthen)
- Everyone wins (users, contributors, founder — positive sum)

**Your role:** Satoshi figure — largest early contributor, not owner. Position from contribution, not control.

**Monetization:** Protocol is free. Services layer monetizes (AI, network access, premium tools).

**References:**
- `projects/teof-ai-product-vision.md`
- `projects/teof-protocol-vision.md`

## Phase 0: MVP Validation (Week 1-2)

**Gate:** Do not build premium until free version validates demand

**Time required:** 1-2 days build, 1-2 weeks observation

### Step 0: TEOF Diagnostic Test (Day 1) — NEW

**Purpose:** Verify TEOF structure functions with LLMs before shipping to users.

**Why this comes first:**
- Governance system failure taught: verify before trusting AI direction
- insights.md Tier 1: "Test Before Build"
- identity.md states diagnostic test is "immediate next step"
- GPT shipping unverified structure repeats the pattern that caused harm

**Method:**
- [ ] Upload TEOF to Claude Project (L1 principles.md + key frameworks)
- [ ] Run realistic queries: relationship advice, investment decision, priority conflict
- [ ] Document: What worked? What failed? Which files retrieved? Response quality?
- [ ] Identify: Are *-core.md files sufficient? Do chapters get retrieved correctly?

**Pass criteria:**
- LLM gives TEOF-aligned responses
- Retrieval works (correct files referenced)
- No hallucination of non-existent principles

**Fail actions:**
- Consolidate files if retrieval fails
- Compress further if context exceeded
- Fix before proceeding to Step 1

**Time:** 1-2 hours

---

### Step 1: Build TEOF Custom GPT (Day 1-2)

- [ ] Create Custom GPT on OpenAI
- [ ] Load L1 principles.md as foundation
- [ ] Load key frameworks (power.md, finances.md, health.md, romance.md)

### Step 3: Observe (Week 1-2)

- [ ] Track: How many users?
- [ ] Track: What do they ask?
- [ ] Track: Do they return?
- [ ] Collect feedback (what's useful, what's missing)
- [ ] Note: What content would explain this better?

**Phase 0 Checkpoint:**

| Metric | Signal | Action |
|--------|--------|--------|
| 100+ conversations | Interest exists | Proceed to Phase 1 |
| Users return | Value delivered | Proceed to Phase 1 |
| Positive feedback | Product-market fit signal | Proceed to Phase 1 |
